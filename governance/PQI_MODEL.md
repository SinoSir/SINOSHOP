# PQI 贡献者模型 / PQI MODEL

> **PQI（Participation Quotient Index）：** 参与贡献指数。  
> 本文件定义从数字层贡献到物理层共建的量化晋升路径。

---

## 1. 模型概述

SINOSHOP 全面开放全球数字仿真协作层。任何人——工程师、科学家、开发者、创作者——
均可通过贡献累积 PQI 分值，获得通往物理共建层的平等通道。

PQI 由算法自动评估，规则公开，数据可审计。

---

## 2. 贡献类别与基础分值

| 类别 | 示例 | 基础 PQI |
|------|------|----------|
| **代码贡献** | PR 被合入（核心模块） | 50–200 |
| **代码贡献** | PR 被合入（工具/脚本） | 20–80 |
| **文档贡献** | 改进或翻译技术文档 | 10–40 |
| **仿真方案** | 提交有效的数字孪生仿真模型 | 30–100 |
| **安全报告** | 报告已验证的安全漏洞（P0/P1） | 100–300 |
| **内容传播** | 技术文章、视频、开源布道 | 5–30 |
| **社区支持** | 解答 Issue、代码审查 | 5–20 |

---

## 3. 架构影响力权重

不同模块的战略价值不同。PQI 基础分值将乘以**架构影响力权重**：

| 模块层级 | 权重 | 说明 |
|----------|------|------|
| Core-Kernel | 3.0 | R16 协议核心、编码逻辑、治理引擎 |
| Core-Specs | 2.5 | 协议标准定义、Schema、校验器 |
| Core-Tools | 1.5 | SDK、仿真工具、CI/CD 脚本 |
| Governance | 1.0 | 治理文档、安全策略 |
| Community | 0.5 | 内容创作、社区运营 |

> ⚠️ **具体权重数值由 SINOSHOP-Core 动态权重引擎计算，此处为公开参考值。**

---

## 4. PQI 晋升路径

| PQI 累计 | 身份 | 权益 |
|-----------|------|------|
| 0–99 | **观察者** | 浏览代码、提交 Issue |
| 100–499 | **贡献者** | PR 合入、参与讨论 |
| 500–1999 | **认证开发者** | 参与核心模块 Review、获得测试环境访问权 |
| 2000–4999 | **核心维护者** | 直接提交至非核心模块、参与技术委员会讨论 |
| 5000+ | **物理共建者** | 获得虎符令牌申请资格，参与物理构件生产与部署 |

---

## 5. 规则与约束

- 所有贡献必须通过已有的治理流水线（版本合规、IP 泄露扫描）方可计入 PQI。
- 任何试图通过刷分、滥用或欺诈手段获取 PQI 的行为将导致积分清零并封禁。
- PQI 积分可通过区块链存证，作为贡献者的永久数字履历。
